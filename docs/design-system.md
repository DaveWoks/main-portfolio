# Portfolio Design System

## Purpose

Define the visual and interaction standards for the portfolio.

Every section, component, and interaction must follow this document.

---

# Design Principles

The design system should feel:

* Intentional
* Structured
* Confident
* Technical
* Premium
* Calm

Avoid:

* Trendy
* Loud
* Decorative
* Flashy

---

# Design Inspiration

Blend inspiration from:

* MONOLOG
* Linear
* Vercel
* Apple

The result should feel:

* Minimal but not empty
* Technical but not cold
* Premium but not flashy
* Elegant but not artistic

# Cards

Cards should:

* Support content
* Not dominate content

---

# Navigation

Navigation should:

* Remain simple
* Remain visible
* Stay unobtrusive

Avoid large navigation systems.

---

# Motion System

## Philosophy

Everything moves.

Nothing dances.

---

# Allowed Motion

Fade

Reveal

Slide

Stagger

Subtle Scale

---

# Avoid

Bounce

Spin

Elastic effects

Attention seeking animation

# Audience Mode Transition

When switching audience mode:

Fade Out

↓

Update Content

↓

Fade In

Duration:

```txt id="jlwmv3"
200ms to 400ms
```

---

# Responsive Rules

Mobile First.

Every component must:

* Work on mobile
* Work on tablet
* Work on desktop

Never design desktop first.

---

# Accessibility Rules

Maintain:

* Keyboard accessibility
* Visible focus states
* Sufficient contrast
* Semantic HTML

Accessibility is mandatory.

---

# Component Standards

Every component should be:

* Reusable
* Composable
* Responsive
* Accessible

Avoid one off components whenever possible.

---

# Success Criteria

The portfolio should feel:

* Premium
* Structured
* Thoughtful
* Confident

Visitors should notice:

The thinking before the visuals.

The content before the effects.

The experience before the technology.
